# 🤝 贡献指南

感谢你对 **BaiduIndexHunter 2.0** 的关注与兴趣！我们欢迎任何形式的建设性反馈和贡献，但请在贡献前确知本项目的非商业性质。

## 📜 行为准则与许可

在参与贡献之前，请务必阅读并遵守 [LICENSE](LICENSE) 协议。

- **严禁商业用途**：本项目的代码仅供学习交流，任何基于此代码的商业行为均属违规。
- **尊重社区**：保持友好、包容和专业的交流态度。

## 🐛 如何报告问题

如果你发现了 bug 或有新功能建议，请通过 GitHub Issues 提交。为了帮助我们更快定位问题，请提供以下信息：

1. **问题类型**：Bug 报告 / 功能建议 / 文档错误
2. **复现步骤**：详细的复现步骤说明，包括使用的指令、配置参数等。
3. **环境信息**：
   - 操作系统（Windows/macOS/Linux）
   - Python 版本（建议 3.11.x）
   - 浏览器版本（Chrome/Edge）
4. **日志截图**：提供相关的报错日志或截图。

## 🛠️ 开发与提交代码

我们欢迎代码贡献，请遵循以下流程：

### 1. 环境准备

请确保你的开发环境符合以下要求：

- **Python**: 3.11.x (必须)
- **Node.js**: 18.x (LTS)
- **MySQL**: 8.0+
- **Redis**: 7.x

### 2. 分支管理

- `main`: 主分支，保持稳定版本。
- `develop`: 开发分支，新功能请基于此分支开发。
- 提交 PR 时，请将源分支命名为 `feature/功能名称` 或 `fix/bug名称`。

### 3. 代码风格

- **后端 (Python)**:
  - 遵循 PEP 8 规范。
  - 使用 4 空格缩进。
  - 尽量添加类型注释 (Type Hints)。
- **前端 (Vue.js)**:
  - 使用 2 空格缩进。
  - 组件命名使用 PascalCase (如 `MyComponent.vue`)。
  - 遵循 ESLint 规范。

### 4. 提交 Pull Request (PR)

1. Fork 本仓库到你的账号。
2. 创建新的分支进行开发。
3. 提交代码前，请确保本地测试通过。
4. 提交 PR 到 `develop` 分支。
5. 在 PR 描述中清晰说明修改内容和解决的问题。

## 📚 文档贡献

文档也是项目的重要组成部分。如果你发现文档有误或需要补充，欢迎直接提交 PR 修改 `README.md` 或其他相关文档。
