If you find a bug or have a feature suggestion, please submit it via GitHub Issues. To help us resolve issues faster, please provide the following information:

1. **Issue type**: Bug report / Feature suggestion / Documentation error
2. **Reproduction steps**: Detailed steps to reproduce, including commands used, configuration parameters, etc.
3. **Environment information**:
   - Operating system (Windows/macOS/Linux)
   - Python version (recommended 3.11.x)
   - Browser version (Chrome/Edge)
4. **Logs or screenshots**: Relevant error logs or screenshots.

## üõ†Ô∏è Development & Submitting Code

We welcome code contributions. Please follow the process below:

### 1. Environment Setup

Ensure your development environment meets the following requirements:

- **Python**: 3.11.x (required)
- **Node.js**: 18.x (LTS)
- **MySQL**: 8.0+
- **Redis**: 7.x

### 2. Branch Management

- `main`: Main branch, stable releases only.
- `develop`: Development branch; base new features on this branch.
- When submitting a PR, name your source branch `feature/feature-name` or `fix/bug-name`.

### 3. Code Style

- **Backend (Python)**:
  - Follow PEP 8 style guide.
  - Use 4-space indentation.
  - Add type hints where possible.
- **Frontend (Vue.js)**:
  - Use 2-space indentation.
  - Use PascalCase for component names (e.g., `MyComponent.vue`).
  - Follow ESLint rules.

### 4. Submitting a Pull Request (PR)

1. Fork this repository to your account.
2. Create a new branch for your changes.
3. Ensure all local tests pass before submitting.
4. Submit a PR to the `develop` branch.
5. Clearly describe your changes and the issues addressed in the PR description.
